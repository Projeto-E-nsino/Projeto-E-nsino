Git é para versionamento de arquivos
 e github é uma plataforma pra hospedar esses arquivos
commit 
merge é a junção de branchs
remote é pra referenciar repositorios
push para colocar o commit no git hub
pull puxa o q está no repositorio do git hub para sua maquina (usado em equipe)
git init - para criar repositorio 
git add - para colocar o nosso arquivo dentro do repositorio
git status - para verificar se deu certo
git commit -m - para falar o nome do comit
git branch -M "main" - para renomear a branch ou passar para outra
git remote add origin + link = para quando eestivermos em outra maquina ou para fazermos conexão do repositorio local e o do github
git push origin main - pra empurrar pro github

git add . = para todos os arquivos
git add (nomedoarquivo) = apenas para esse arquivo

git checkout -b "" = serve para criar uma nova branch e já sai da branch atual e indo pra dentro dessa nova branch.
git checkout (nome da branch) = para ir pra outra branch
git merge (nome dabranch) - para juntar uma branch a principal --> em seguida usar o git push

git clone + link - puxar arquivos de outras pessoas e trazer pra minha maquina

cd + nome do arquivo - para entrar na pasta
clear - limpar o gitbash
git pull - para atualziar arquivos q vc clonou, ou alterações q foram feitas por voce ou outras pessoas
Fork - serve para trazer o repositorio de outra pessoa para o seu perfil
pull requests - serve para mandar uma alteração q outra pessoa fez (mesmo q n tenha permissão)
